Overview

StockCRM is the operating system I built for small Moroccan retail businesses to run inventory, sales, customers and finance from one place. It started as a stock tracker and grew into a small SaaS — multi-tenant, role-based, with its own billing layer.

The interesting parts aren't the CRUD. They're the AI order agent that reads messages and handwritten ledgers, the auto-generated landing pages, the public storefront API for partner shops, and a manual-payment billing model that fits how Moroccan SMBs actually pay.

The Four Things

01

ERP, but tight

Products, orders, customers, suppliers, expenses, ledger, deliveries — the full SMB ops surface, with snapshot pricing, low-stock alerts and per-product profitability.

02

AI order entry

Partners message in plain language — or send a photo of a handwritten daily ledger — and an agent drafts the order. Two-phase confirm with HMAC-signed state, so nothing hits the DB until a human approves.

03

Multi-tenant from row zero

Every row scoped by organizationId. JWT auth with rotating refresh tokens, four roles, fifty-plus granular permissions, and subscription expiry checked on every request.

The AI Order Agent

Highlight

A partner sends a message — or a photo of a handwritten daily ledger — and the agent drafts an order. Nothing is written until a human confirms.

01Draft

Agent parses the message, runs tool calls (product search, customer match), returns a draft order. The state is HMAC-signed and held client-side — no DB write yet.

02Confirm

Human approves. Server verifies the signature, then writes the order. Tampering between phases is detectable and rejected.

Ledger photo parsing

Horizontal lines separate orders; the agent extracts product + price (MAD) per row.

Refusal recovery

Generic “cannot assist” replies get rephrased and retried with stricter tool guidance.

Loop safeguards

Iteration cap and a forced-finalize fallback prevent infinite tool ping-pong.

Audit prefix

Agent-created orders are tagged AGNT- so ops can trace what the AI did.

Decisions Worth Calling Out

01

Two-phase agent flow with signed state

The model can iterate, search products, ask follow-ups — without leaving orphan rows. The signature catches tampering between draft and confirm.

02

Manual-payment SaaS billing

Customers in Morocco pay by bank transfer. Skipping Stripe kept the codebase honest and the customers happy.

03

Org header, not subdomain

x-organization-id makes multi-org users a per-request decision instead of a forced re-login.

04

AGNT- prefix on AI orders

Tiny audit signal. Ops can answer “what did the AI do today?” with one LIKE query.
