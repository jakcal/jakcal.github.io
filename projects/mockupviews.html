Overview

MockupViews is a single-page web app for creators, indie devs and marketers who need polished device mockups, App Store screenshots and short promo animations — without booting Photoshop or fighting native screen-recorders.

Two complementary workflows live under one editor. Mockup mode drops screenshots inside realistic frames with backgrounds, 3D camera and effect stacks. AppStore mode batch-produces store-listing assets at fixed Apple and Google sizes. A keyframe-based timeline turns either of them into a video, rendered in the browser by FFmpeg.wasm.

The Four Things

01

Zero-server pipeline

Every pixel is composed client-side. DOM-to-canvas rendering for stills, FFmpeg.wasm for video. No upload, no backend, no quotas — your screenshots never leave the tab.

02

Two workflows, one engine

Mockup mode places shots inside realistic device frames; AppStore mode batch-builds store-listing assets at fixed Apple/Google sizes. The mode flag swaps panels and canvas while shared state survives.

03

Animation timeline in a still tool

Keyframed camera and transform tweening with linear and eased interpolation. RAF-driven scrubbing, then exported to MP4 or WebM through an in-browser FFmpeg.wasm encoder.

04

Performance discipline

Slice-based Zustand stores, low-res preview cache while editing, blob-URL lifecycle management, Web Worker exports, IndexedDB-backed undo history with a 50-step ring buffer.

The Render Pipeline

Highlight

A still-image tool that exports MP4 — the trick is doing it all client-side. The pipeline has three stages, none of them touch a server.

01Compose

Editor renders the canvas as a live React tree — frames, screenshots, effects, text overlays.

02Snapshot

modern-screenshot serializes the DOM to canvas at the chosen scale. Per-frame for animations.

03Encode

FFmpeg.wasm runs in a Web Worker, stitching frames into MP4 / WebM with a progress callback.

Blob lifecycle

URLs created lazily, revoked after export — memory stays flat across long sessions.

Performance Mode

Full-res images swapped for cached low-res variants while editing; full quality only at render.

Phase-aware UI

ExportPreviewPlaceholder replaces the canvas during render with badge, ETA and cancel.

Off-thread encode

FFmpeg.wasm and the snapshotter run in a worker, RAF playback stays smooth.

Decisions Worth Calling Out

01

Client-only render pipeline

DOM → canvas → WASM video. No backend means no upload latency, no per-user storage cost, and the privacy story writes itself.

02

Slice-based Zustand store

The editor store split into image / canvas / device / download domains, plus 10+ feature stores. Selectors everywhere — one slider drag doesn't re-render the timeline.

03

Mode flag, not separate apps

Mockup vs AppStore swap the canvas + side panels through a single editorMode flag. Camera, history, fonts, watermark stay shared — switching modes never costs the user state.

04

Phase-aware export overlay

ExportPreviewPlaceholder replaces the canvas during render with an animated badge, ETA, and cancel control. The UI stays alive while FFmpeg churns off-thread.
